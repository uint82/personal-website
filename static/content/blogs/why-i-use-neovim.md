Everyone's got a take on editors. Mine is Neovim. Here's why? and it's probably not what you expect.

## It's not about speed

The first thing people say when defending Neovim: "It's blazing fast." Sure. But VS Code isn't slow. Zed exists. If raw startup time was my actual bottleneck, I'd have bigger problems.

Speed is a nice bonus, not the reason.

## It's not about minimalism either

I use Hyprland and Arch, so people assume I'm allergic to anything that isn't a 2KB binary. Not really. I care about efficiency, not purity. If a tool makes me more productive, I use it.

## The actual reason: it fits my brain

Vim motions aren't just shortcuts. They're a language. `ci"` means "change inside quotes." `dap` means "delete around paragraph." Once that language is in your muscle memory, editing text feels like *thinking out loud*. You stop managing your editor and start managing your ideas.

Every other editor I've tried puts me in a mode where I'm navigating the UI. In Neovim, I'm navigating the *content*. That's a fundamental difference.

## The config is the feature

Yes, Neovim takes time to configure. That's not a bug. When I set up my config, I had to think about what I actually needed. No defaults to hide behind. No bloated feature set to ignore.

The result is an editor that does exactly what I want and nothing else. I know what every plugin does because I chose it.

## Should you use it?

Probably not, if you're asking. The learning curve is real, and VS Code with a Vim plugin gets you 80% of the way there. If that 80% is enough, take it.

But if you've ever felt like your editor is in your way like you're fighting the tool instead of the problem maybe give it a real shot.

Not for the speed. For the language.
